Pytorch学习

快速入门

一、加载数据

pytorch有两个库与数据处理相关：torch.utils.data.DataLoader和torch.utils.data.Dataset。

Dataset存储了样本和它对应的标签。

PyTorch提供特定于领域的库，如TorchText，TorchVision和TorchAudio， 所有这些都包括数据集。在本教程中，我们将使用TorchVision数据集。该模块包含许多真实世界视觉数据的对象，例如 CIFAR，COCO。在本教程中，我们使用 FashionMNIST 数据集。每个TorchVision的Dataset包含两个参数：transform和target_transform来分别修改样本和标签。

将Dataset作为参数放入DataLoader。这将在我们的数据集上包装一个可迭代对象，并支持自动数据加batching, sampling, shuffling 和multiprocess。在这里，我们定义一个 64 的批大小，即每个元素在数据加载器中，可迭代将返回一批 64 个特征和标签。

二、创建模型

为了在 PyTorch 中定义神经网络，我们创建了一个类继承 nn.Module。我们在__init__函数定义网络的各层 ，并在__forward__函数中指定数据如何通过函数中的网络。要加速在神经网络中的操作中，我们将其移动到 GPU 或 MPS（如果可用）。

三、优化模型参数

为了训练模型，还需要一个损失函数和优化器。

在单个训练循环中，模型对训练数据集进行预测（批量馈送到它），并且 反向传播预测误差以调整模型的参数。

我们还根据测试数据集检查模型的性能，以确保它正在学习。

训练过程通过多次迭代（epoch）进行。在每个时期，模型学习 参数以做出更好的预测。我们打印模型在每个时期的准确性和损失;我们希望看到 精度随着每个时期的增加而增加，损失减少。

Tensor类似与Numpy数组，Tensor能在Gpu和其他硬件上加速。。事实上，张量和 NumPy 数组通常可以共享相同的底层内存，无需复制数据。张量还针对自动微分进行了优化。

三、操作张量

超过100个张量运算，包括算术，线性代数，矩阵操作（转置， 索引、切片）、采样等。

这些操作中的每一个都可以在 GPU 上运行（速度通常高于 中央处理器）。

默认情况下，张量是在 CPU 上创建的。我们需要使用 方法显式地将张量移动到 GPU 中（在检查 GPU 可用性之后）。

Dataset和DataLoader

PyTorch 提供了两个数据基元：torch.utils.data.DataLoader和 torch.utils.data.Dataset允许您使用预加载的数据集以及您自己的数据。 存储样本及其相应的标签，并环绕可迭代对象 以便轻松访问样品。

一、加载数据集

下面是如何从TorchVision加载Fashion-MNIST数据集的示例。 Fashion-MNIST是Zalando文章图像的数据集，由60000个训练示例和10000个测试示例组成。 每个示例都包含一个 28×28 灰度图像和来自 10 个类之一的相关标签。

我们使用以下参数加载 FashionMNIST 数据集：

  • root：是存储训练/测试数据的路径，
  • train：指定训练或测试数据集，
  • download=True：从互联网下载数据（如果数据在 root上不可用）
  • transform并指定特征和标注变换target_transform

四、使用数据加载程序进行训练

检索数据集的特征并一次标记一个样本。在训练模型时，我们通常希望 在“小批量”中传递样本，在每个时期重新洗牌数据以减少模型过度拟合，并使用 Python 的 加快数据检索速度。DataLoader是一个迭代对象，它在一个简单的 API 中为我们抽象了这种复杂性。
