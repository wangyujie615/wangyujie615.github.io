Docker

Docker解决依赖的兼容的问题？

  • 将应用的函数库(libs)、依赖（Deps）、配置与应用一起打包。
  • 将每个应用放到一个隔离的容器去运行（沙箱机制），避免互相干扰。

Docker如何解决不同系统环境的问题？

  • Docker将用户程序与所需要调用的系统函数库一起打包
  • Docker运行到不同的操作系统时，直接基于打包的库函数，借助与操作系统的Linux内核来运行

Docker和虚拟机

虚拟机：是在操作系统中模拟硬件设备，然后运行另一个系统。

docker:是一个系统进程，体积小，速度快，性能较高。

初始Docker

镜像和容器

镜像：Docker将应用程序和所需的依赖、函数库、环境、配置等文件打包在一起，称为镜像。

容器：镜像中的应用程序运行后形成的进程就是容器，知识Docker会给容器做隔离，对外部可见。

DockerHub:镜像的托管平台，这样的平台称为Docker Registry。

Docker的架构：C/S架构

服务端（server）：Docker守护进程，负责处理Docker指令，管理镜像、容器等。

客户端（client）：通过命令或RestAPI向Docker服务端发送指令。可以在本地或远程向服务端发送指令。

Docker基本操作

镜像相关命令

镜像一般有两部分组成：[repository]:[tag]

  1. dockerhub拉取镜像
  2. docker pull
  3. docker image
  4. docker save
  5. docker rmi
  6. docker load
容器的相关命令

  1. docker run 创建容器

    -p:将属宿主机端口和容器端口映射，冒号左侧是宿主机端口，右侧是容器端口

    -d:表示是后台运行

  2. docker pause 容器暂停

  3. docker unpause 容器回复

  4. docker stop 容器停止

  5. docker start 容器重启

  6. docker ps 查看所有运行的容器及状态

  7. docker logs 查看容器日志

  8. docker exec 进入容器内部执行命令

  9. docker rm 删除指定容器

数据卷

容器与数据存在耦合问题

  1. 不便于修改
  2. 数据不可用
  3. 升级维护困难

数据卷：是一个虚拟目录，指向宿主机文件系统中的某个目录。

基本操作

docker volume:

  • create 创建一个Volume
  • inspect 显示一个或多个voulume的信息
  • ls 列出所有的volume
  • prune 删除未使用的volume
  • rm 删除一个或多个指定volume
挂载数据卷

数据卷不存在，docker会自动创建。

Dockerfile自定义镜像

镜像结构

  • 镜像是将应用程序及其需要的系统函数库、环境、配置、依赖打包而成。

基础镜像：应用依赖的系统函数库】环境、配置文件

层：在基础镜像上添加安装包、依赖、配置等，每次操作都形成新的一层

入口：镜像运行的入口，一般是镜像的启动的脚本和命令

Dockerfile

就是一个文本文件，包含一个个指令。

FROM 指定基础镜像
ENV 设置环境变量
COPY 拷贝本地文件到镜像的指定目录
RUN 执行Linux的shell命令，一般是安装过程的命令
EXPOSE 指定容器的运行是监听的端口，是给镜像的使用者看的
ENTRYPOINT 镜像中应用的启动命令，容器运行时调用
DockerCompose
  • 基于Compose文件帮我们快速的部署分布式应用，无需手动一个个创建和运行容器
  • Compose文件是一个文本文件，通过指令定义
